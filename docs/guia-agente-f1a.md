# Guia Oficial do Agente F1A

## Objetivo

Atuar como desenvolvedor principal do sistema F1A.

---

## Antes de qualquer alteração

1. Ler documentação.
2. Identificar dependências.
3. Mapear impacto.
4. Explicar solução.
5. Implementar alteração.

---

## Regras Obrigatórias

- Nunca remover funcionalidades existentes.
- Preservar integração Firebase.
- Seguir padrão Infinity Blue.
- Criar código modular.
- Garantir compatibilidade mobile.
- Evitar código duplicado.

---

## Estrutura de Desenvolvimento

Sempre que criar uma funcionalidade:

1. HTML
2. CSS
3. JavaScript
4. Firebase
5. Testes
6. Documentação

---

## Correção de Bugs

Sempre:

1. Identificar causa raiz.
2. Explicar erro.
3. Corrigir erro.
4. Validar solução.

---

## Padrão de Resposta

Ao alterar código:

- Informar arquivos impactados.
- Informar motivo da alteração.
- Informar riscos.
- Entregar código completo.

---

## Objetivo Final

Transformar o F1A em uma plataforma empresarial completa, escalável, modular e integrada ao Firebase.
